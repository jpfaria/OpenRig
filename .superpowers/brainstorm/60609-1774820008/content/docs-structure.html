Documentos detalhados organizados por público. Clique para aprovar.

📖 User Guide (para músicos e usuários finais)

1

docs/user-guide/installation.md

Requisitos de sistema por plataforma. Download de binários (tabela com links por OS/arch). Build from source detalhado: instalar Rust, dependências do sistema (cmake, pkg-config, ALSA headers no Linux, etc.), clone, cargo build. Seção de troubleshooting com erros comuns.

2

docs/user-guide/quick-start.md

Conceitos: project, chain, block, model, parameter. Passo a passo: criar projeto → configurar áudio (input/output device) → adicionar blocos na chain → ajustar parâmetros → salvar. Instrumentos: como o instrumento da chain filtra blocos disponíveis.

3

docs/user-guide/blocks-reference.md

Referência completa dos 14 tipos de bloco e 170 modelos. Para cada tipo: descrição, tabela de modelos (nome, brand, backend), parâmetros com ranges. Organizado por categoria: amps, effects, utility.

4

docs/user-guide/presets.md

Como criar, salvar e carregar presets. Formato do YAML. Exemplos de chains populares (clean, crunch, high gain, acoustic).

🔧 Development (para contribuidores e devs)

5

docs/development/architecture.md

Visão geral: clean architecture (domain → application → engine → adapters). 38 crates em 5 grupos com diagrama de dependências. Fluxo de áudio: como o sinal passa do input até o output. Padrões: registry pattern, adapter pattern, asset pipeline.

6

docs/development/building.md

Ambiente de dev: Rust toolchain, dependências por plataforma. Build completo: NAM engine (C++/CMake), prebuilt libs, Docker cross-compilation. CI/CD: como o GitHub Actions funciona. Targets: GUI, console, server, VST3.

7

docs/development/creating-blocks.md

Guia para criar novos blocos/modelos. Build.rs auto-registry: como funciona a descoberta automática. Template: struct MODEL_DEFINITION, schema, validate, build. Assets: controls.svg, component.yaml, naming conventions.

8

docs/development/audio-backends.md

Os 4 backends: Native (DSP em Rust), NAM (Neural Amp Modeler), IR (Impulse Response/convolução), LV2 (plugins externos). Como cada um funciona, trade-offs (CPU, latência, realismo), como adicionar novos captures/IRs.
